• What is Templeet ?

    Templeet is an open-source project initiated by Pascal Courtois and Fabien Penso. It allows you to create photo galeries, news systems, personal sites, blogs, etc. Using Templeet, you can generate HTML, CSS, SVG pictures, SMIL, and any kind of text files.

    Templeet's strong points are ease of install, performance, and increased productivity through multiple levels of cache. When all caches are activated, performance is similar to serving static pages, ie. around 400 hits/second on a PII 400Mhz processor.

    Templeet, is distributed under the free Gnu Public License (GPL) and open source. The latest source code is available through CVS, and discussions about development can be held in the public mailing lists. We do welcome user feedback and additions.

    The documentation is currently pretty thin and is mostly only available in French, but it should be enough to get you started... provided you do read French, of course. We'd be grateful to any kind soul who could take the time to translate it into English (note: it gets better, so you can still take a look).

    Only a few examples of templates are provided, but they should be good enough to give you a rough idea of what you can do with Templeet. If you need help, members of the mailing lists could provide support in a timely manner. You can also take a look at the available packages, and if this is your first Templeet installation, we invite you to read the documentation.

    Templeet only needs a web server (Apache) and PHP. It supports databases but none is needed to run Templeet.

  • Features

    Here are some of Templeet's characteristics :
    • Template system
    • Full caching system. Near static pages performances
    • Multilingual management compatible with Apache's MultiView
    • Full authentication management (using either: file, MySQL or PostgreSQL). Other API are possible.
    • Package system, install and update
    • Automatic thumbnail generation
    • Works with Unix and Windows
    • Full independent installer, no need for an Internet connection while installing
    • Fitted for small personnal sites as well as huge sites serving millions of hits per day

      Trying Templeet is keeping it !

      When it was about creating a site with PHP two main possibilities were offered to the webmaster: using a whole made system like all the flavours of "nuke", or doing everything oneself.

      However, in my eyes the "nukes" present many drawbacks: personnalisation is poor (and we get a site looking like thousands of others), their use is adequate for information sites but not for other types of sites, their deployment is very heavy and we finish with a mamoth even when using only few functionalities !

      While coding everything from nothing is a huge amount of work which also suppose fair technical skills for you wish to propose interesting functionalities.

      I first used Templeet to create my own personnal site. I admit I was puzzled by syntax in the beginning, but after a day all was clear. With PHP the content is integrated within the code which lessens readability; with Templeet you can call functions directly from the content which is much more intuitive and clear when rereading.

      After that first convincing experiment I went in the creation of two other sites of bigger dimensions. Thanks to Templeet I could do in three days a new version of a site I had done in three weeks! Templeet offers a whole set of "classical" functions ready to use. No more need to code everything "by hand". Among these functions I especialy appreciated authentication management and users' rights, multilingual handling and communication with data bases. I managed at last to make sites perfectly conforming to XHTML.

      In the end I must admit I became addicted to Templeet. For futur realisations I won't imagine any more working without it ! Trying Templeet is keeping it, you'll be warned...

      wap, webmaster of Heartical.net et LesBancsPublics.fr.
